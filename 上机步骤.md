# 上机步骤

本项目用于 Unitree Go1 + Slamtec/Slamware 2D 激光雷达的实时 2D SLAM 实验。实验输入不是离线数据集，而是 Slamware ROS SDK 实时发布的 `/scan`，类型必须是 `sensor_msgs/LaserScan`。

## 1. 硬件与网络

- Go1 机器人 IP：`192.168.31.11`
- 雷达和 Go1 通过网线直连，不需要额外配网
- 通过 NoMachine 远程进入 Go1 的 Ubuntu 系统

## 2. 上机前确认

确认 Go1 上已有：

- Ubuntu 18.04
- ROS1 Melodic
- Python2.7
- Slamware ROS SDK 2.8.2_rtm 对应工作空间
- 本项目整个目录 `slamware_loop_slam_project`

如果现场断网，先安装离线依赖：

```bash
cd ~/slamware_loop_slam_project/offline_deps/python2_pip
bash install_offline_python2_deps.sh
```

如果 `python2 -m pip` 不存在，先用离线 `.deb` 安装 `python-pip`，或者优先用 apt/deb 安装 `python-numpy`、`python-scipy`、`python-rospkg`、`python-catkin-pkg`。

## 3. 编译本项目

```bash
source /opt/ros/melodic/setup.bash
cd ~/slamware_loop_slam_project/catkin_ws
chmod +x src/slamware_loop_slam/scripts/*.py
catkin_make
source devel/setup.bash
```

## 4. 按官方方式启动 Slamware

先启动官方 server node：

```bash
source /opt/ros/melodic/setup.bash
source ~/UnitreeSLAM/catkin_lidar_slam_2d_go1/devel/setup.bash
roslaunch slamware_ros_sdk slamware_ros_sdk_server_node.launch ip_address:=192.168.31.11
```

另开一个终端，用官方 view launch 打开 RViz：

```bash
source /opt/ros/melodic/setup.bash
source ~/UnitreeSLAM/catkin_lidar_slam_2d_go1/devel/setup.bash
roslaunch slamware_ros_sdk view_slamware_ros_sdk_server_node.launch
```

## 5. 检查 `/scan`

```bash
rostopic list | egrep 'scan|odom|map'
rostopic type /scan
rostopic hz /scan
rostopic echo -n 1 /scan
```

`rostopic type /scan` 应输出：

```text
sensor_msgs/LaserScan
```

只有确认 `/scan` 正常后，再启动自己的 SLAM 节点。

## 6. 启动无回环基础版

如果官方 Slamware 节点已经启动，只启动本项目算法：

```bash
cd ~/slamware_loop_slam_project/catkin_ws
source /opt/ros/melodic/setup.bash
source devel/setup.bash
roslaunch slamware_loop_slam scan_icp.launch
```

输出：

- `/slamware_icp_pose`
- `/slamware_icp_path`
- `catkin_ws/src/slamware_loop_slam/result/no_loop_path.csv`

## 7. 启动回环 + 位姿图优化版

```bash
cd ~/slamware_loop_slam_project/catkin_ws
source /opt/ros/melodic/setup.bash
source devel/setup.bash
roslaunch slamware_loop_slam pose_graph_slam.launch
```

输出：

- `/slamware_local_pose`
- `/slamware_local_path`
- `/slamware_pg_pose`
- `/slamware_pg_path`
- `catkin_ws/src/slamware_loop_slam/result/pose_graph_local.csv`
- `catkin_ws/src/slamware_loop_slam/result/pose_graph_optimized.csv`
- `catkin_ws/src/slamware_loop_slam/result/pose_graph_loops.csv`
- `catkin_ws/src/slamware_loop_slam/result/pose_graph_keyframes.csv`

## 8. 一条命令整合启动

如果你希望本项目同时 include 官方 Slamware server node：

```bash
roslaunch slamware_loop_slam slamware_no_loop.launch ip_address:=192.168.31.11
roslaunch slamware_loop_slam slamware_with_loop.launch ip_address:=192.168.31.11
```

注意：如果官方 Slamware server 已经单独启动，就不要再用这两个整合 launch，避免重复启动。

## 10. 调参建议

回环版优先跑默认参数。若轨迹突然跳变，先看终端里是否出现 `ICP error`、`translation jump`、`rotation jump`。若回环一直为 0，可以逐步调大：

- `loop_candidate_distance`
- `loop_descriptor_threshold`

若误回环较多，收紧：

- `loop_icp_error_threshold`
- `loop_max_measurement_translation`
- `loop_max_measurement_rotation_deg`
